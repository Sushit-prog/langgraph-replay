# Plan: Add pytest-llm semantic blame integration

## Context
The blame engine (`src/langgraph_replay/blame.py`) already has a MODE 2 semantic blame section (lines 141-163) that imports `pytest_llm`, calls `assert_regression`, and checks `.passed`. The user wants to change this to catch `AssertionError` instead and directly blame the node with `confidence="high"`, using keyword arguments.

## Changes

### 1. Modify `src/langgraph_replay/blame.py` (lines 141-163)

Replace the existing MODE 2 semantic blame block. Current code uses positional args and checks `.passed`; new code uses keyword args and catches `AssertionError`.

**Before:**
```python
if pytest_llm and baseline_execs:
    baseline_by_name = {e.node_name: e for e in baseline_execs}
    if execution.node_name in baseline_by_name:
        baseline_exec = baseline_by_name[execution.node_name]
        baseline_output = _deserialize_state(baseline_exec.output_state)
        for key, val in output_state.items():
            if key.startswith("_"):
                continue
            if isinstance(val, str) and len(val) > 20:
                baseline_val = baseline_output.get(key, "")
                if isinstance(baseline_val, str) and len(baseline_val) > 20:
                    try:
                        result = pytest_llm.assert_regression(
                            val, baseline_val, threshold=0.75
                        )
                        if not result.passed:
                            issues.append(
                                f"Semantic regression on key '{key}': "
                                f"similarity {result.similarity:.2f} < threshold"
                            )
                    except Exception as e:
                        logger.debug(f"pytest-llm check failed: {e}")
```

**After:**
```python
if pytest_llm and baseline_execs:
    baseline_by_name = {e.node_name: e for e in baseline_execs}
    if execution.node_name in baseline_by_name:
        baseline_exec = baseline_by_name[execution.node_name]
        baseline_output = _deserialize_state(baseline_exec.output_state)
        for key, val in output_state.items():
            if key.startswith("_"):
                continue
            if isinstance(val, str) and len(val) > 20:
                baseline_val = baseline_output.get(key, "")
                if isinstance(baseline_val, str) and len(baseline_val) > 20:
                    try:
                        pytest_llm.assert_regression(
                            output=val,
                            baseline=baseline_val,
                            threshold=0.75,
                        )
                    except AssertionError as e:
                        blamed = execution
                        confidence = "high"
                        reason = str(e)
                        issues.append(f"Semantic regression: {e}")
```

Key differences:
- Uses keyword args `output=`, `baseline=`, `threshold=`
- Catches `AssertionError` (not generic `Exception`)
- When caught: sets `blamed`, `confidence`, and `reason` directly (early blame)
- No longer checks `.passed` / `.similarity` — relies on the exception
